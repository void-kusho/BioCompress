# Wire format issue:
# Encode: returns buffer with [4-byte length at offset 0, bitstream at offset 16]
# Compress: wraps encode output in 12-byte header -> [12-byte header][4-byte length][bitstream]
# Decompress: strips 12-byte header, passes [4-byte length][bitstream] to huffman_decode()
# Decode: reads num_symbols from offset 0, calculates byte_idx = bit_pos/8 = 16/8 = 2
#   -> reads from input[2] but that's still part of the length prefix!
# Root cause: bit_pos starts at 16 but input only has the length prefix at bytes 0-3
</context>

  <action>
Fix the output size calculation in huffman_encode(). Current code:
```c
size_t out_size = (num_symbols * 2 + 7) / 8 + 1;
```

Replace with correct calculation that accounts for:
- 4 bytes for the length prefix
- Actual bitstream size (num_symbols * 2 bits, rounded to bytes)

```c
size_t bitstream_size = (num_symbols * 2 + 7) / 8;
size_t out_size = 4 + bitstream_size;  // 4-byte length prefix + bitstream
```

Also fix the bit_pos starts at 16 bits (2 bytes) but we should start at bit 32 (after the 4-byte length prefix). The code currently starts at bit_pos = 16, which skips over the first 2 bytes of the length prefix incorrectly.

Actually the simplest fix: Change bit_pos from 16 to 32 since the 4-byte prefix is at bytes 0-3, which is bits 0-31. Start writing at bit 32.
